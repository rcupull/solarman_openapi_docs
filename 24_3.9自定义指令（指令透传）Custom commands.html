3.9 Custom commands (command transparent transmission) Custom commands

View on Yuque
Introduction
This interface corresponds to the custom command function of the SOLARMAN platform device. It essentially transmits the command transparently to the device and returns the original response result of the device to you.
This interface corresponds to the custom instruction function of the SOLARMAN platform device. Essentially, the instruction is passed through to the device, and the result of the device response is returned to you
System flow System flow
When you encounter a problem, you can first use the custom command function of the SOLARMAN platform to test the device. For example, after the instructions can be sent and received normally through the SOLARMAN platform, the API can be used to automate the implementation.
When you run into problems, you can first test the device using the custom instruction feature of the SOLARMAN platform. For example, after sending and receiving instructions normally through the SOLARMAN platform, it can be automatically realized by using API.


Interface details

Notice:
1Only after your device has customized command functions on the Wheat Merchant Edition platform or Wheat Smart Platform can you perform API layer operations through this interface.
2The maximum number of requests per minute for a single OpenApi account to call the interface: 50
3This interface only provides transparent transmission channels. Please contact the device manufacturer for specific instructions.
custom directive
Interface description
Supports the issuance of custom transparent transmission instructions through OpenApi. Through this interface, developers can encapsulate command services by themselves to meet the needs of rich personalized command scenarios.

The response parameters in the callback result are defined below
Callback address request type: POST
After sending the command, the command result will be requested every 3 seconds for the first 60 seconds, and the command result will be requested every 30 seconds after 60 seconds. Once the command result is available, a callback will be initiated. The callback party needs to return a "SUCCESS" response, otherwise the callback will be reinitiated. If the callback cannot succeed after more than 600s, it will be discarded and no callback will be made.
