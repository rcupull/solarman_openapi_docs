timeType=2:Statistical data with daily dimensions
Statistics with a daily dimension
startTimeandendTimeUse yyyy-MM-dd format, no more than 30 days
The input parameter values ​​are the same, which means obtaining the data of the current day.
startTime and endTime are in yyyy-MM-dd format with a maximum of 30 days
The value of the input parameter is the same, which means that the data of the day is obtained

timeType=3: Statistical data with monthly dimension
Statistics on a monthly basis
startTimeandendTimeUse yyyy-MM format, no older than 12 months
The input parameter values ​​are the same, which means obtaining the data of the current month.
startTime and endTime are in yyyy-MM format and not older than 12 months
The value of the input parameter is the same, which means that the data of the current month is obtained

timeType=4: Statistical data with year as dimension
Statistics in terms of years
startTimeandendTimeUse yyyy format
The input parameter values ​​are the same, which means obtaining data for the current year.
startTime and endTime use the yyyy format
The value of the input parameter is the same, which means that the data of the current year is obtained

timeType=5: Get the data frame within the specified 24 hours through UTC time
Retrieve frames from the specified 24 hours in UTC
startTimeandendTimeUse 10-digit second-level unix timestamp (passed in string format)
endTime - startTime <= 86400
startTime and endTime use 10-bit second-level unix timestamps (passed in string format) that do not exceed 24 hours
