A1: I have a power station on the Wheat Platform. Why can’t I find the power station through OPENAPI?
Q: If your power plant is on the Wheat merchant version, please use the merchant version token to obtain the power plant data; if your power plant is on the Wheat Smart Electric, please use the smart electric token to obtain the power plant data.
The difference between the merchant version token and the smart electronic token is: If the orgId (merchant ID) is entered when obtaining the token, the merchant version token is obtained; if the orgId (merchant ID) is not entered when obtaining the token, the smart electronic token is obtained.

A2: How do I get the orgId (merchant ID)?
Q: Please contact after-sales service to obtain it directly, or query interface 2.2 to obtain it.

A4: Why can't I control the device through OPENAPI?
Q: Before controlling through OPENAPI, please confirm that you have device control permissions on the Wheat Merchant Edition or Wheat Smart Platform. If you do not have device control rights, please contact the device manufacturer for authorization first.
If you have device control rights:
If you are authorized to use "custom instructions", please use interface 3.9 for control. Please consult the equipment manufacturer for the content (command details) that need to be entered in the interface;
If you are authorized for "single command", control through API is not supported at the moment.
If you are authorized for "batch instructions", control via API is not currently supported.

A5: The interface call returns a device not found error. What is the reason?
Q: The reason is that the device you are querying is not under the account corresponding to your API call. Please log in to the Wheat platform (merchant version/smart electronics) to confirm whether this device is under your account, and then confirm whether the account used to change the device is the same as the account used when calling the API.

A6: The interface call returns "invalid param type", how to adjust it?
Q: It may be caused by a variety of reasons. You need to compare each of your input parameters with the instructions in this document for confirmation. If you cannot confirm, please provide the details of the specific interface you called, input parameter details, and error report details. Send an email to customerservice@solarmanpv.com for troubleshooting. (It is also best to provide the APPID and the account you are calling)

A7: The interface call returns "AUTH_INVALID_USERNAME_OR_PASSWORD". What is the reason?
Q: It may be caused by a variety of reasons: The API domain name used by the interface is incorrect (for accessing domestichttps://api.solarmanpv.comFor visiting other countrieshttps://globalapi.solarmanpv.com）
Or the account information is wrong (only one of the three account forms of email, mobile phone, username can be selected as input parameters. The parameter name username or email needs to match the actual one. If it is a mobile phone account, the countryCode parameter needs to be added)
Or the password is wrong (you need a password that can log in to the Mai platform, and it is encrypted with SHA256 and is lowercase)
Or the calling format (Param and body) is wrong.
If it cannot be checked, please provide the interface call details (url, input parameter details) to contact after-sales for troubleshooting.

A8: Why can’t the data on the Wheat platform be queried through OPENAPI?
Q: Except for a few functions that do not support OPENAPI calls, most data on the Wheat platform can be queried through OPENAPI: and the data found by OPENAPI is completely consistent with the Wheat platform.
If you are not sure which interface a certain data on the Wheat Platform corresponds to, please provide a screenshot of the data on the Wheat Platform and contact after-sales service to provide the interface number;
If you think the data on the Wheat platform is inconsistent with the data obtained by OPENAPI, please provide screenshots of the data on the Wheat platform, detailed screenshots of the data being called by OPENAPI, and provide after-sales feedback.
